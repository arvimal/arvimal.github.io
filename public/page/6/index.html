Custom CRUSH rulesets and pools

xxxx published on 2015-09-23 included in ceph
Ceph supports custom rulesets via CRUSH, which can be used to sort hardware based on various features such as speed and other factors, set custom weights, and do a lot of other useful things. Pools, or the buckets were the data is written to, can be created on the custom rulesets, hence positioning the pools on specific hardware as per the administrator’s need. A large Ceph cluster may have lots of pools and rulesets specific for multiple use-cases.

OSD information in a scriptable format

xxxx published on 2015-09-18 included in ceph
In case you are trying to get the OSD ID and the corresponding node IP address mappings in a script-able format, use the following command: # ceph osd find This will print the OSD number, the IP address, the host name, and the default root in the CRUSH map, as a python dictionary. # ceph osd find 2 { “osd”: 2, “ip”: “192.168.122.112:6800\/5311”, “crush_location”: { “host”: “node4”, “root”: “default”}} The output is in json format, which has a key:value format.

Monitor maps, how to edit them?

The MON map is used by the monitors in a Ceph cluster, where they keep track of various attributes relevant to the working of the cluster. Similar to the CRUSH map, a monitor map can be pulled out of the cluster, inspected, changed, and injected back to the monitors, manually. A frequent use-case is when the IP address of a monitor changes and the monitors cannot agree on a quorum.

Calculate a PG id from the hex values in Ceph OSD debug logs

xxxx published on 2015-08-30 included in ceph programming python
Recently, I had an incident where the OSDs were crashing at the time of startup. Obviously, the next step was to enable debug logs for the OSDs and understand where they were crashing. Enabled OSD debug logs dynamically by injecting it with: # ceph tell osd.* injectargs –debug-osd 20 –debug-ms 1 NOTE: This command can be run from the MON nodes. Once this was done, the OSDs were started manually (since it were crashing and not running) and watched out for the next crash.
