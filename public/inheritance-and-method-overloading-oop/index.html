Because of Inheritance, the functions/methods defined in parent classes can be called in Child classes which enables code reuse, and several other features. We have two classes, the parent class beingÂ MyClassÂ and the child class beingÂ MyChildClass.

MyClassÂ has three methods defined.

  • my_set_method()
  • my_get_method()

The printdoc()Â method is anÂ Abstract method, and hence should be implemented in the Child class as a mandatory method.

The child class MyChildClassÂ inherits from MyClassÂ and has access to all it’s methods.

Normally, we can just go ahead and use the methods defined in MyClassÂ , in MyChildClass. But there can be situations when we want to improve or build upon the methods inherited. As said earlier, this is calledÂ Method Overloading.

MyChildClassÂ extends the parent’s my_set_method()Â function by it’s own implementation. In this example, it does an additional check to understand if the input value is an intÂ or not, and then calls the my_set_method()Â of it’s parent class using super. Hence, this method in the child class extends the functionality prior calling method in the parent. A post on superÂ is set for a later time.

Even though this is a trivial example, it helps to understand how the features inherited from other classes can be extended or improved upon via method overloading.

The my_get_method()Â is not overridden in the child class but still called from the instance, as instance_1.my_get_method(). We’re using it as it is available via Inheritance. Since it’s defined in the parent class, it works in the child class’ instance when called, even if not overridden.

The printdoc()Â method is an abstract method and hence is mandatory to be implemented in the child class, and can be overridden with what we choose to do.

Inheritance is possible from python builtins, and can be overridden as well. Let’s check out another example:

How does the code work?

The class MyList()Â inherits from the builtin list. Because of the inheritance, we can use list’s available magic methods such as __getitem__()Â , __setitem__()Â etc..

NOTE: In order to see the available methods in list, use dir(list).

  1. We create two functions/methods named `__getitem__()` and `__setitem__()` to override the inherited methods.
  2. Within these functions/methods, we set our own conditions.
  3. Wie later call the builtin methods directly within these functions, using
    1. list.__getitem__()
    2. list.__setitem__()
  4. We create an instance named xÂ from MyList().
  5. We understand that
    1. x[1]Â and x.__getitem__(1)Â are same.
    2. x[4, 'e'] and x.__setitem__(4, 'e')Â are same.
    3. x.append(f)Â is same as x.__setitem__(<n>, f)Â where is the element to the extreme right which the python interpreter iterates and find on its own.

Hence, in Inheritance, child classes can:

  • Inherit from parent classes and use those methods.
    • Parent classes can either be user-defined classes or buitins like listÂ , dictÂ etc..
  • Override (or Overload) an inherited method.
  • Extend an inherited method in its own way.
  • Implement an Abstract method the parent class requires.
